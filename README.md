# Multi-Vendor E-Commerce Platform

A full-stack marketplace built with **Spring Boot** (backend) and **React** (frontend), following the blueprint exactly.

---

## Tech Stack

| Layer     | Technology                        |
|-----------|-----------------------------------|
| Backend   | Spring Boot 3.2, Spring Security  |
| Auth      | JWT (jjwt 0.11.5)                 |
| Database  | MySQL 8+                          |
| Frontend  | React 18, React Router v6, Axios  |

---

## Project Structure

```
ecommerce/
├── backend/
│   └── src/main/java/com/ecommerce/
│       ├── config/          # SecurityConfig (CORS, JWT, roles)
│       ├── controller/      # AuthController, ProductController, CartController, OrderController, AdminController
│       ├── dto/             # AuthDto, ProductDto, CartDto, OrderDto
│       ├── entity/          # User, Product, Category, Cart, CartItem, Order, OrderItem
│       ├── exception/       # GlobalExceptionHandler + custom exceptions
│       ├── repository/      # JPA repositories with pagination & custom queries
│       ├── security/        # JwtUtil, JwtAuthFilter, CustomUserDetailsService
│       └── service/         # AuthService, ProductService, CartService, OrderService
└── frontend/
    └── src/
        ├── components/      # Navbar, Pagination, ProtectedRoute
        ├── context/         # AuthContext (global user state)
        ├── pages/           # Login, Register, Home, ProductDetail, Cart, Orders, VendorDashboard, AdminOrders
        └── services/        # api.js (axios + interceptors), authService, productService, cartService, orderService
```

## Database Setup

```sql
CREATE DATABASE ecommerce_db;
```

The schema is auto-generated by Hibernate (`spring.jpa.hibernate.ddl-auto=update`).

---

## Backend Setup

### 1. Configure credentials

Edit `backend/src/main/resources/application.properties`:

```properties
spring.datasource.url=jdbc:mysql://localhost:3306/ecommerce_db
spring.datasource.username=root
spring.datasource.password=yourpassword

jwt.secret=404E635266556A586E3272357538782F413F4428472B4B6250645367566B5970
jwt.expiration=86400000
```

### 2. Run

```bash
cd backend
mvn spring-boot:run
```

Backend starts on `http://localhost:8080`.

## API Reference

### Auth
| Method | Endpoint         | Access  | Description       |
|--------|-----------------|---------|-------------------|
| POST   | /auth/register  | Public  | Register user     |
| POST   | /auth/login     | Public  | Login, get JWT    |

### Products
| Method | Endpoint                    | Access   | Description                     |
|--------|-----------------------------|----------|---------------------------------|
| GET    | /products                   | Public   | List approved products (paginated, searchable) |
| GET    | /products/{id}              | Public   | Get single product              |
| GET    | /vendor/products            | VENDOR   | Get own products                |
| POST   | /vendor/products            | VENDOR   | Create product (pending approval) |
| PUT    | /vendor/products/{id}       | VENDOR   | Update own product              |
| DELETE | /vendor/products/{id}       | VENDOR   | Delete own product              |
| PUT    | /admin/products/{id}/approve| ADMIN    | Approve product                 |

### Cart
| Method | Endpoint              | Access    | Description         |
|--------|-----------------------|-----------|---------------------|
| GET    | /cart                 | CUSTOMER  | View cart           |
| POST   | /cart/add             | CUSTOMER  | Add item to cart    |
| DELETE | /cart/remove/{id}     | CUSTOMER  | Remove cart item    |

### Orders
| Method | Endpoint        | Access    | Description                    |
|--------|----------------|-----------|--------------------------------|
| POST   | /orders         | CUSTOMER  | Place order from cart          |
| GET    | /orders         | CUSTOMER  | Get own orders (paginated)     |
| GET    | /orders/vendor  | VENDOR    | Get orders with vendor's items |
| GET    | /admin/orders   | ADMIN     | Get all orders                 |

## Role-Based Access

| Role     | Permissions                                                    |
|----------|----------------------------------------------------------------|
| CUSTOMER | Browse products, manage cart, place orders, view own orders    |
| VENDOR   | CRUD own products, view orders containing their products       |
| ADMIN    | Approve products, view all orders                              |

---

## Key Implementation Details

- **JWT**: Stateless authentication — token validated on every request via `JwtAuthFilter`
- **DTOs**: Entities never exposed directly; all responses go through DTO mapping
- **@Transactional**: Used on `placeOrder` (stock deduction + cart clear) and `addToCart` to ensure atomicity
- **Pagination**: All list endpoints accept `page`, `size`, `sortBy` query params; return Spring `Page<T>`
- **Search**: `/products?search=keyword` triggers `LIKE` query on product name
- **Product approval**: New/updated products set `approved=false` until admin approves
- **Price snapshot**: `OrderItem.price` stores price at time of order — not affected by future product price changes
- **Global exception handling**: All errors return structured `{ message, status }` JSON — no stack traces exposed
